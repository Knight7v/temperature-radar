# 7天预测页面重新设计

**项目:** 空调热销区域雷达  
**页面:** 7天预测  
**日期:** 2025-05-11  
**类型:** 功能增强

---

## 1. 需求概述

### 1.1 当前问题
现有"7天预测"页面使用简单的表格展示聚合数据（未来3天ADI、7天最高ADI等），无法满足用户查看**未来7天逐日趋势**的需求。

### 1.2 设计目标
重新设计为**"未来7天空调热销趋势矩阵"**，让用户能够：
1. 快速判断哪些城市未来7天持续高温
2. 识别夜间温度持续偏高的城市
3. 发现ADI正在上升的城市
4. 预测哪一天可能进入行动级或爆发级

---

## 2. 数据结构扩展

### 2.1 新增数据字段

扩展 `Core.generateCityData()` 生成的城市对象，新增 `future7Days` 数组：

```javascript
{
  city: "北京市",
  region: "华北",
  province: "北京",
  lat: 39.92,
  lng: 116.46,
  
  // 现有字段保持不变...
  
  // 新增：未来7天每日数据
  future7Days: [
    {
      date: Date对象,           // 该天的日期
      dayMax: number,            // 白天最高温
      nightMin: number,          // 夜间最低温
      dayFeelsLike: number,      // 白天体感温度
      nightFeelsLike: number,    // 夜间体感温度
      humidity: number,          // 湿度
      adiScore: number,          // 当日ADI
      level: string,             // 当日等级
      isHotNight: boolean        // 是否热夜
    },
    // ... 共7个元素（索引0 = 今天）
  ]
}
```

### 2.2 生成逻辑

**Core.generate7DayDailyData(city, dayIndex)** - 新增函数
- 基于前一天数据渐进式变化
- 温度变化范围：-2°C 到 +3°C
- 湿度变化范围：-5% 到 +5%
- 确保数据连续性和合理性

## 4. 三个维度展示

### 4.1 维度1：高低温度（默认）

**单元格内容：**
```
最高温 / 最低温
36° / 31°
```

**颜色规则：**
- 最高温：≥35°C深红，33-34°C橙色，30-32°C黄色
- 最低温：≤25°C深青，26-27°C浅青，≥28°C红色

**热夜标记：**
- 夜间最低温 ≥ 28°C：右上角显示 "🌙"

---

### 4.2 维度2：高低体感温度

**单元格内容：**
```
最高体感 / 最低体感
40° / 33°
```

**颜色规则：**
- 最高体感：≥40°C深红，37-39°C橙色，34-36°C黄色
- 最低体感：≤28°C深青，29-31°C浅青，≥32°C红色

**热夜标记：**
- 夜间体感 ≥ 30°C：显示 "🌙"

---

### 4.3 维度3：ADI指数

**单元格内容：**
```
96 🔴
```

**背景色规则：**
- ADI ≥ 90：深红 #D0021B
- 85-89：红色 #F57C00
- 75-84：橙色 #F5A623
- 65-74：黄色 #FBC02D
- 55-64：蓝色 #4A90E2
- < 55：灰色 #666666

## 5. 筛选与搜索

### 5.1 筛选功能（保持现有按钮，调整逻辑）

**筛选选项：**
- 全部、爆发级、行动级、机会级、观察级
- 夜间闷热（未来7天有≥2天热夜）
- 未来升温（第7天ADI比今日高≥10分）
- 华东、华南、华中、西南、华北、西北、东北

**筛选逻辑：**
- 单选（保持现有交互）
- 点击筛选按钮 → 更新 `UI.state.reportFilter` → 调用 `renderReport()`

---

### 5.2 搜索功能（新增）

**搜索框位置：**
- 页面顶部右侧，标题旁边
- 样式：浅色背景，圆角，带搜索图标
- 宽度：200px

**搜索行为：**
- 实时搜索（输入时即时过滤）
- 搜索范围：城市名称
- 大小写不敏感

**无结果处理：**
- 显示 "未找到匹配的城市"
- 提供"清空搜索"按钮

## 6. 排序规则

**默认排序：**
- 按**未来3天ADI均值**从高到低
- 均值相同时，按**今日ADI**从高到低

## 10. 边界情况与性能

### 10.1 数据异常处理

- 缺失数据：显示 "-"
- 计算失败：使用默认值
- `future7Days` 数组长度不足7：按实际天数渲染，不显示缺失天数的列

### 10.2 性能优化

- 考虑虚拟滚动（可选）
- 搜索防抖300ms
- 按需渲染可见区域

### 10.3 响应式设计

- 宽度 < 1400px：横向滚动
- 宽度 < 768px：隐藏部分固定列
